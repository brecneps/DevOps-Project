Welcome to My DevOps Project!



Introduction

Hello! Welcome to my DevOps principles project, in this website that I have created I will go over the steps I took to create a CI/CD pipeline with AWS, and how it applies to the DevOps principles listed below. This is a website I have built using html and css in vscode. I have hosted it in an S3 Bucket using an AWS CodeBuild Pipeline to pull its source from my GitHub repository and automatically build, test, and deploy with every push made to update the website. On top of this I have configured CloudFront Metrics as the CDN allowing users from all over the globe to reach this unique url quickly and efficiently. I have also configured some extra security alarms outside of the core security that CloudFront brings with WAF, these were configured with CloudWatch where I have created a dashboard for these metrics. Images are linked and shown without links down below and provide further comments as well.


Continuous Integration

This website is hosted on an S3 bucket that is using an AWS CodeBuild pipeline to automatically pull its source from my GitHub repository, which builds itself using a yaml file I configured. The file sets the pipepline up for handing off the proccess to its testing phase- allowing for errors to be detected before deployment. After testing successfully then deploying automatically, this allows for continuous integration through its source of its GitHub link allowing for anything integrated, whether its features or fixes to the website to appear quickly and continuously.


Continuous Delivery

All code pushed into the main branch of the source GitHub repository for this website is continually delivered automatically any time there is a new push to it, via the CodeBuild pipeline that is connected to the S3 bucket this site is hosted on. This creates a flow of continuous delivery for any user requesting this page. This structure allows for any feature or fix to this website to be streamlined into an efficient delivery system that allows for any release to be delivered globally, through the other connected sevice of CloudFront making solving issues a quick process once the fix has been found.


Monitoring and Logging

This website is using CloudWatch to monitor alarms that I have configured with certain metrics such as a high request rate alarm for total request reaching over 100 in five minutes indicating a potential DDoS attempt, in pair with this is a high error rate alarm, alarming only when 4xxErrors in at least 15 instances exceeds more than 20 errors within 1 hour and 15 minutes. Both are connected to the S3 bucket and held on a dashboard in CloudWatch allowing for monitoring, if these are alarmed I will be sent an email through Amazon's SNS which then logs this in my email. This in pair is just an added protection on top of the standard CloudFront has with WAF. Another service I am using is CloudFront which is logging aspects like what devices, browsers, operating systems and locations are requesting this page. These are helpful metrics when troubleshooting on a larger scale website. I am also using a trial of Pingdom for other useful metrics and to receive a text if the site goes down.


Communication and Collaboration

Through GitHub and comments made in this html file, there is a layer created of communication and collaboration, as I am able to leave detailed comments about changes I have made or plan to make in my commits. Others are then able to see this and merge their code with mine. I am able to do the same with theirs, adding to the layer of collaboration this structure creates a teamwork and communication centered enviroment with progress only moving forward via communication and collaboration as without it conflicts are created and problems emerge.
